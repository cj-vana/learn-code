# Content Authoring

Content lives under `content/python` as original YAML. Default catalog loads only `review_status: published`. Required review flow: briefed, drafted, reviewed_originality, reviewed_pedagogy, reviewed_correctness, tests_validated, published.

Paths (`kind: path`) live under `content/python/paths/` and reference existing published item ids in ordered units. `learn-code validate-content` errors on unknown or duplicate item references and warns when an item precedes the items that teach its prerequisite concepts. The seed paths are generated by `scripts/generate_paths.py`; re-run it after adding a content area.
